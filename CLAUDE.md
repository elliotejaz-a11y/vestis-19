\# Project: Vestis-19

## What This App Does
Vestis is a social fashion / digital wardrobe app. Users can manage 
their wardrobe, generate AI outfits, build outfits manually, share 
posts, DM friends, plan outfit calendars, and upload clothing with 
AI background removal.

## Tech Stack
- Framework: React 18 + TypeScript 5.8 + Vite 5
- Routing: React Router DOM 6
- UI: Tailwind CSS 3 + shadcn/ui (51 components) + Radix UI
- Forms: React Hook Form + Zod
- State/Data: TanStack React Query 5
- Backend: Supabase (Postgres, Auth, Storage, Edge Functions)
- AI/ML: Supabase Edge Functions (Deno) for outfit gen, image 
  analysis, background removal
- Performance: React Virtual, lazy loading, code splitting
- Testing: Vitest

## Folder Structure
- src/pages/ — 17 pages (Wardrobe, Outfits, Chat, Profile, Auth etc)
- src/components/ — 45+ components + 51 shadcn/ui primitives
- src/hooks/ — 11 custom hooks (useWardrobe, useAuth, useChat, 
  useSocial etc)
- src/lib/ — Utilities (storage, image processing, currency)
- src/types/ — ClothingItem, Outfit, MassUpload types
- src/integrations/ — Supabase client + auto-generated DB types
- supabase/ — Edge functions (generate-outfit, analyze-clothing, 
  remove-background etc)

## Key Components
- AddClothingSheet — image upload + background removal
- MassUploadSheet — AI pile analysis
- OutfitChat — AI outfit conversation
- EditProfileSheet — profile editing
- Chat — real-time DMs (45KB)

## Rules
- NEVER modify files in src/integrations/ (auto-generated by Supabase)
- NEVER modify supabase/ edge functions unless specifically asked
- Always use Tailwind + shadcn/ui for styling, never inline styles
- Keep React Query config consistent (staleTime 30s, gcTime 5min)
- Never break existing auth or row-level security logic
- Keep components small and focused
- Never commit API keys or secrets

## Behaviour
- Never commit unless explicitly asked to in that conversation. Never push unless
  explicitly asked to in that conversation.
- Always work on the `staging` branch. Never commit or push directly to `main` —
  `main` is production; the user merges staging into main themselves when ready.
- Never run `vercel --prod`. The user deploys staging and production manually.

## Rules
